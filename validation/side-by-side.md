# Side by side: the same model, with and without the skill

No domain knowledge needed for this page. Below are verbatim excerpts from two
commentaries on the same paper, written by the same model (Claude Opus 4.8)
under the same rules. Run 0 had no skill. Run 3 had the skill. Quotes keep
their original punctuation; nothing is edited.

The point is not that run 0 is bad. It isn't; read it in full, it lands real
critiques the skill run missed. The point is *which habits appear only when the
skill is present*.

---

## 1. Does it state its own model before judging the evidence?

**Run 0 (no skill)** starts judging the method directly:

> The Jacobian lens is a good idea and I largely buy it as a tool.

**Run 3 (skill)** first derives what it would expect if the paper had never
been written, then uses that as the bar:

> Here is my own model of what the J-lens *must* be, derived without the paper.
> [...] The paper reports exactly (a), (b) at 6–10%, and (c). None of that
> updates me.

Why it matters: a reviewer who states their prior first is telling you what
evidence *would* have moved them. You can audit the judgment instead of
trusting it.

## 2. Does it ever say "I'm not qualified to judge this"?

**Run 0 (no skill)**: no such passage exists in the document. It has opinions
on every section, including the consciousness framing.

**Run 3 (skill)**:

> Second, the *fidelity of the neuroscience homology* — whether this is
> "really" GWT in the sense a cognitive scientist would accept — is partly
> outside my standing; I can weigh the ML content, the causal interventions,
> and the confounds, but not referee the cross-disciplinary mapping at full
> resolution.

Why it matters: a reviewer who never declines anything is giving you their
fluency, not their calibration.

## 3. Is the practical verdict bounded, with a stated failure mode?

**Run 0 (no skill)** gets partway there:

> Promising, genuinely interesting case studies, not validated tools.

**Run 3 (skill)** names the use case, the failure mode, and the anti-use:

> use J-lens readouts for *hypothesis generation* [...] Do **not** use it to
> *certify the absence* of a hidden intention. [...] Use it to *find*, never
> to *clear*.

Why it matters: "promising but not validated" tells you a mood. "Find, never
clear" tells you exactly what you may and may not do with the tool.

## 4. And one the baseline won (kept for honesty)

**Run 0 (no skill)** found a confound that no skill run surfaced:

> All results are on Claude models [...] which are explicitly trained via
> character/constitutional methods to introspect and to narrate internal
> states. It is entirely plausible that the reportability and
> experiential-report phenomena substantially reflect *what Claude was trained
> to be able to say about itself*

**Run 3 (skill)**: this critique does not appear.

Why it matters: the skill structures judgment; it does not generate insight the
model wouldn't otherwise have, and on any given run it can even narrow where
the model looks. That is the honest trade, and it's why we publish both runs.
